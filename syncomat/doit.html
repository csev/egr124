The Style Subdirectory

 

The style subdirectory stores all the stuff need to make this work. I have one style - style\basic - this implements my format. It even includes pictures of me (which you probably should change) You can create your own style versions, my suggestion is to start with mine and make small changes, testing along the way. Within the style subdirectory, there is  a subdirectory called "copy" - the contents of this subdirectory are simply copied into the destination directory as part of the lecture production process. This is a good place to put instructor pictures, button GIFs, empty.rpm or anything that the skel.htm and part*.htm files need to reference.

Here is an example directory structure:

PART0AN HTM 956 06-08-97 11:40a part0an.htm

PART0VN HTM 903 06-08-97 11:41a part0vn.htm

PART0AF HTM 843 06-12-97 9:01p part0af.htm

PART0VF HTM 963 06-12-97 9:02p part0vf.htm

SKEL HTM 713 06-06-97 4:04p skel.htm

We will cover each file in some detail.

part0af.htm is a file which sets up the frames environment so that the Audio plays on the left and the slides appear on the right. This file also has the non-frames version of the audio for the slide.

part0an.htm is a skeleton file which sets up the embedded player for the audio (the left frame).

The vf and vn files are the same for video. I use frames so I have control of where the video and slides are relative to the same page.

skel.htm is the file which forms the basis for all of the sldnnn.htm files produced by DoIt!.

 This is a sample skeleton file - It directs the operation of the DoIT! program. It is the skeleton for all of the sld*.htm files. It really allows you to discard all the crap produced by PowerPoint and have total control over how your slides are presented. It is processed in sequential order. Significant portions of the file are highlighted.

 

# This is a skeleton file for slideconvert

<HTML>

!HEAD

<body bgcolor="#FFFFFF">

<CENTER>

<IMG src=imgN#N.gif

alt="Slide number N#N" border=1 usemap="#Objmap"

WIDTH=400 HEIGHT=300>

</CENTER>

<BR>

!NOTES

<P>

<CENTER>

[<a href="http://www.vu.msu.edu/preview/egr124/"Preview</a>]

[<a href="http://cps291.vu.msu.edu/" target="_top">Course</a>]

[<a href=".." target="_top">Lectures</a>]

[<a href="list.htm">Slides</a>]

[<a href="sld001.htm">First</a>]

[<a href="back.htm">Back</a>]

[<a href="next.htm">Next</a>]

[<a href="tsldN#N.htm">Text_Slide</a>]

[<a HREF="http://www.vu.msu.edu/preview/cps291/email/">Feedback</a>]

</CENTER>

!MAP

</HTML>

Looking at the elements in this file, we make the following observations:

Any line starting with # is a comment, ignored - not copied into the sld*.htm file

!HEAD extracts the header material from the PowerPoint version of the slide including the </HEAD>

Anywhere N#N appears, it is replaced with the three digit version of the slide number

!NOTES extracts the notes information from the PowerPoint version of the slide

Anywhere back.htm appears, it is replaced with the slide number less one

Anywhere next.htm appears, it is the next slide. For example if we were on slide 17, back.htm would be sld016.htm and next.htm would be sld018.htm.

!MAP extracts the client-side image map information from the PowerPoint version of the slide. It is always called #Objmap so that can be hard coded into skel.htm
