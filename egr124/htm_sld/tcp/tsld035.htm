Notes:

    Each system which uses the Domain Name System consults a pre-configured Domain Name Server to “revolve” its domain name addresses into IP addresses. For most dial-up connections, the Domain Name Server is set automatically. To tace a request, assume that a user on scully.msu.edu connects to www.holt.k12.mi.us (rewmember that the http:// indicates how to connect and the www.holt.k12.mi.us indicates where to connect)
    (1) Scully consults its configured domain name server (35.9.37.10). The domain name server for egr.msu.edu checks its local table
    (2) and does not find www.holt.k12.mi.us - it has a list of the top-level domain name servers, so it contacts the top-level domain name server for .us
    (3). The .us server returns the address of mi.us to the egr server which then contacts the mi.us server
    (4). The mi.us server returns the address of the k12.mi.us server
    (5). This continues until the domain name server for holt.k12.mi.us consults it table
    (6) to find the actual IP address of www.holt.k12.mi.us and returns it to the egr server (7).
    Finally (8) the egr server returns the IP address to scully.msu.edu which then makes a direct IP connection (9) to the www.holt.k12.mi.us using its IP address (204.38.68.2).
